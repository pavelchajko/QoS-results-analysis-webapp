Custom Filtering

As the name indicates, this feature allows to preform custom queries on the results sets. After selection of attributes and dimensions of interest we get values satisfying the condition.

Note: dimension ranges are pre-set on satisfying scores.

Completeness Misiing

Estimates the number of missing values of the analyzed attribute records. Value is set to 0 if record is not present, or 1 if it is present.
(used to see if the data source is corrupted, so we can know which records to exlude in data processing.)

Completeness Frequency

Estimates the ratio between an efective frequency of a value over the expected frequency of that value.So, it makes a forecast based on a previous analysis that took place in a different time interval. As close to 1 it means the percentage of error of the estimation is lower.
(It is also computed in the case of missing attribute values in the source.)

Volume

The Volume quality dimension is used to discover the number of occurrences of a specific value (of the selected attribute) in the data source. Use this filter to understand which values are the most frequent one.
(For example for attribute CODLINHA - find the most frequent line of bus.)

Consistency

We use it when we want to verify if a combinations of events that has been observed during time can be proved to be a rule of the data source. Rule is a combinations of events that happen in the data source if its conditions are satisfied.
(For example a passenger that takes the most used bus line takes also another bus line in the same day. It is a good idea to combine it with Volume so first we can find a frequent record)

Timeliness

Giving the user understanding of the current data in a timely manner

Precision

It is defined as the degree to which repeated measurements show the same or similar results, and it means that all the values should be equally distributed and not too distant between each other.
It gives a good approximation of how the aggregated values are close to each other
